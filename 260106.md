- **시스템 아키텍쳐 정리**:
1. 물리적 네트워크 구조 (Network Topology)

모든 장치는 동일한 라우터(STA 모드) 하위망에 존재하며, 서로 고유 IP를 통해 자유롭게 통신합니다.

    중앙 관리자 (노트북 A): VS Code를 통해 각 에지 노트북에 원격 접속하여 코딩 및 모니터링 수행.

    에지 노드 (노트북 B, C, D): 각 거점에 고정되어 로봇과 1:1로 통신 및 제어 수행.

    물리 로봇 (RoboMaster EP): 에지 노드의 명령을 받아 구동하고 센서 데이터를 전송.

2. 논리적 소프트웨어 아키텍처 (Layered Architecture)

Layer 1: 중앙 지능 및 가상화 (노트북 A)

    Unity Digital Twin: 모든 로봇의 위치를 실시간 시각화.

    AI Engine (ML-Agents): 가상 환경에서 학습된 모델을 기반으로 최적 경로/행동 결정.

    Master Logic: 로봇과 에지 노드(B, C, D) 사이의 거리 기반 1:1 매칭 계산.

    Tool: VS Code Remote Extension, ROS-TCP-Endpoint.

Layer 2: 분산 통신 및 실행 (노트북 B, C, D)

    Edge Controller: 담당 로봇의 상태를 수집하고 명령을 전달하는 게이트웨이.

    ROS 2 Node: /odom(위치) 발행 및 /cmd_vel(명령) 구독.

    Docker Container: 하드웨어 의존성 없는 일관된 실행 환경 제공.

Layer 3: 하드웨어 (RoboMaster EP)

    Sensors: IMU, 오도메트리, 거리 센서 데이터를 에지로 전송.

    Actuators: 에지로부터 전달된 속도/방향 명령 수행.
